理解基线：

元素内容，和元素的位置确认的一条看不见的线。

影响单一元素基线的CSS声明

  • 字体大小（font-size）的变化，是指字体大于默认字体（1rem, 16px）
  • 元素的行高（line-height）也会影响基线的位置。随着行高的不断增加，基线会在原来的位置向下偏移。

    • 如果元素包含子元素（行高 = 元素的高度）

    说明：基线的作用范围，就是子元素所在的行。

  • 子元素（display: inline / inline-block）时，如果子元素的字体大小（font-size）的变化，是指字体大于默认字体（1rem, 16px），父元素的基线会改变，向下偏移。
  • 如果子元素行高（line-height）不断增加，父元素的基线会在原来的位置向下偏移。
  • 同一行的基线由字体最大，或者行高最大的元素决定。
  • inline-block 没有文本内容，父元素会在基线处开始呈现元素，例如img.如果需要此类元素正好位于父元素的Y轴的1/2处，我需要给该元素设置：vertical-align: middle.
  • Inline-block 有内容：如果没有给该元素设置行高，它使用的是父元素的行高，会导致其自身的文本内容“溢出”。同时该子元素会在其行高的顶部“渲染”。

    • 谁导致基线变化

    前提条件：父元素的行高（line-height）和高度（height）相同。

  • 1、元素的字体大小基于默认字体增大时，会导致元素的基线发生变化，向下移动。
  • 2、如果父元素包含子元素（inline-block）无文本内容，或者自闭合标签（img），会在父元素基线开始呈现子元素。
  • 3、如果父元素包含子元素（inline-block）有文本内容，会以文本的基线开始呈现元素。
  • 4、Inline，inline-block元素的基线就是其父元素的基线。
  • 5、inline，inline-block元素的行高改变，会导致父元素的基线改变。
  • 6、inline，inline-block元素字体大小的变化，会导致父元素基线发生变化。
  • 7、block元素字体的变化，行高的变化，不会导致父元素基线发生改变。

    • 自闭合标签的基线

  • img元素，以及没有内容元素：父元素会从基线处渲染该元素。
  • 表单输入元素：元素的中间

    • 必须保证子元素都再一行。

  • 如果一个父元素包含了多个子元素，基线会以行高最大元素的基线，作为他们的基线。
  • 如果一个父元素包含了多个子元素，某个子元素的字体大小发生改变，他们的基线也会发生改变。
